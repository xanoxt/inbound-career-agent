# Плейбук 07 — Еженедельный дайджест

> **Когда запускать:** раз в неделю (или по запросу). Главный клей системы — связывает остальные плейбуки в один ритм.
> **Язык:** русский. **Режим:** human-in-the-loop (только черновики).
> **Входы:** `profile/positioning.md`, `market-config.md`, свежие артефакты из `output/` (watchlist, входящие возможности, черновики контента).

---

## Цель

Собрать неделю в один короткий RU-документ: что появилось на рынке, что пришло входящим, какой контент готов, и **не больше 3 действий** на этой неделе. Пользователь решает, что делать — агент ничего не отправляет сам.

## Выход

`output/YYYY-MM-DD-digest.md` (RU). Структура:

```md
# Дайджест за неделю — YYYY-MM-DD

## ⭐ Действие этой недели (максимум 3, по приоритету)
1. [конкретное действие] — почему важно
2. ...
3. ...

## 📡 Рынок (дельта с прошлой недели)
- Новые компании/роли, созвучные позиционированию (из watchlist)
- Движение по зарплатным якорям вашего уровня

## 📥 Входящие (нужно квалифицировать)
- Список запросов/сообщений, требующих ответа → плейбук 02

## ✍️ Контент готов к публикации
- Черновик(и) из плейбука 03 — на ваш одобрение

## 🎯 Разговоры в работе
- Собеседования/звонки, где нужна подготовка → плейбук 05

## ❓ Открытые вопросы к вам
- Что требует вашего решения
```

## Правила

1. **Верх документа — действия.** Пользователь должен видеть их, не скролля. Максимум 3, с причиной.
2. **Не выдумывать.** Каждая строка должна опираться на артефакт в `output/` или `profile/`. Нет данных — секция пустая с пометкой «на этой неделе без изменений».
3. **Фильтровать по alignment.** В раздел «Рынок» и «Входящие» попадает только то, что прошло alignment-чек (см. раздел 12 `positioning.md`). Остальное — в один пункт «отсеяно: N».
4. **Никогда не применять, не публиковать, не писать от себя.** Только предлагать.
5. В конце спросить: «Какое из действий делаем на этой неделе?»
