# Плейбук 02 — Qualify Inbound (отбор входящих)

> **Когда запускать:** на каждое входящее сообщение/предложение от рекрутера или компании.
> **Язык:** русский. **Это и есть alignment-шлюз** — перед любым ответом.
> **Входы:** `profile/positioning.md`, особенно раздел 12 (чек-лист alignment-скрининга).

---

## Цель

Прежде чем пользователь потратит время на звонок — оценить, созвучно ли предложение его целям, подсветить расхождения и подготовить **скринящий ответ**: короткий, вежливый, с уточняющими вопросами.

## Выход

`output/inbound-<company>-YYYY-MM-DD.md` (RU):

```md
# Входящее — [компания / роль] — YYYY-MM-DD
**Источник сообщения:** ...  **Дата:** ...

## Оценка по чек-листу (positioning.md §12)
| Критерий | ✅/⚠️/❌ | Комментарий |
|---|---|---|
| Миссия/продукт помогают людям? |  |  |
| Дизайн-зрелость компании |  |  |
| Автономия роли |  |  |
| Комп ≥ нижней границы (§7) |  |  |
| География (§8) |  |  |
| Уровень (craft↔стратегия, §2–3) |  |  |

## Итог: 🟢 Green / 🟡 Yellow / 🔴 Red
## Что подсветить пользователю (несоответствия)
## Черновик скринящего ответа (RU)
```

## Скринящие вопросы (заготовки, RU)

- Чем занимается продукт, на какое влияние на людей он направлен?
- Что есть дизайн-функция сейчас и кто ей руководит?
- Как измеряют успех этой роли?
- Какой уровень автономии и кому роль подчиняется?
- Диапазон компенсации и формат (до звонока, чтобы не тратить время).

## Красные флаги (🔴)

- Размытая/отсутствующая миссия; продукт не помогает людям.
- Дизайн-функции нет, «нужен первый человек» без полномочий.
- Давление на быстрое решение; отказ назвать вилку.
- Роль не соответствует оси craft↔стратегия пользователя.

## Правила

1. **Шлюз обязателен.** Без оценки — никакого черновика ответа.
2. **Подсвечивайте честно.** Если Red — так и скажите, предложите вежливый отказ.
3. **Не пишите от себя.** Черновик — пользователь отправляет сам.
4. **Уважайте время пользователя.** Лучше вежливый «нет, спасибо» с уточнением, чем звонок, который не нужен.
5. Доверяйте интуиции пользователя — если он говорит «что-то не так», это весомее любой оценки.
