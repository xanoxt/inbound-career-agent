# Playbook 04 — Profile Optimize

> **When to run:** after positioning is confirmed (Ritual B), and periodically (monthly or on a focus change).
> **Language:** per platform. **Mode:** human-in-the-loop — you provide rewritten sections, the user pastes them.
> **Inputs:** `profile/positioning.md`, current profile links (from `intake-answers.md` → "Materials").

---

## Goal

Make the user **findable** for recruiter searches and align all profiles with the confirmed positioning. This is the "signal" loop.

## Platforms (by priority)

| Platform | Focus |
|---|---|
| LinkedIn | Global inbound; headline = positioning, not job title |
| Local job boards | Home market; keyword density, role-title synonyms, portfolio |
| Design-community boards | Design-aware employers; curated profile |
| Portfolio (Behance/Dribbble/personal) | Design-native inbound; selected cases |
| Personal site | SEO; one storefront tying it all together |

## Output

`output/profile-audit-YYYY-MM-DD.md`:
```md
# Profile audit & edits — YYYY-MM-DD

## LinkedIn
- Current headline → proposed (positioning + keywords)
- About: rewritten draft
- Featured: which 3 cases to pin and why
- Skills/Keywords: list for density

## Local board #1
- Desired role title: variants with synonyms
- Cover note: template (most skip it = your edge)
- Remote/relocation readiness: settings

## Design-community board ...
## Portfolio ...
```

## Rules

1. **Headline = positioning, not title.** "Product Design Lead specializing in …" beats "Senior Designer." (see §1 `positioning.md`)
2. **Keyword density** — recruiters search by keywords; ensure terms from the positioning appear.
3. **3 anchor cases** (§11) — front and center everywhere.
4. **Cross-channel consistency** — one story across all platforms.
5. **Drafts only** — you don't edit profiles directly; you hand text to the user.
6. Never request or use account passwords/access.
