## 3. Методология зарплатного якоря

Принцип: **НЕ один источник** (малые выборки, смещения). Шаги:

1. Собрать диапазоны из ≥2 источников (getmatch + Habr-Lead + hh).
2. Учесть **уровень**: Lead/Senior — выше средних цифр по профессии.
3. Читать регион из `profile/` (по умолчанию СПб); Москва обычно +5–10%.
4. Различать **пол vs цель**: средняя по профессии ≠ целевая для лида.

Ориентир (generic product designer, dreamjob.ru, 2026) `[уточнить live]`:
- средн. ~200 000 ₽/мес на руки; диапазон 170–230 тыс.; min 150, max 260.
- top-плательщики: Домклик (260), Т-Банк (230), Яндекс (200).
- **Это ПОЛ, не цель для Lead.** Для лида целевая — выше; точную вилку получать агрегатом по уровню.

## 5. Карта inbound-каналов (RU)

Где дизайнер-лид становится находимым:
- **getmatch.ru, Habr Карьера** — профили под поиск рекрутерами.
- **VC.ru, Habr-блог** — контент-маркетинг работодателям (плейбук 03).
- **Behance / Dribbble / личный сайт** — дизайн-native inbound + SEO.
- **Telegram-сообщества** — UX/дизайн-чаты; культура TG-first после первого контакта.
- **Выступления** — UX Russia, Conf Talks и т. п. (для лида — design-leadership повестка).
- **LinkedIn (EN)** — для глобального/remote inbound (VPN; ToS caveat).

## 6. Сигналы к отслеживанию (раз в неделю)

- Новый найм design-лидов в целевых архетипах.
- Появление новых продуктов/стартапов в целевом сегменте (помогающие продукты).
- Движение зарплатных вилок уровня пользователя.
- Активность целевых компаний в блогах/медиа (VC.ru, Habr).

## 7. Замена рынка (для другого региона)

Рынок и язык ортогональны. Чтобы адаптировать под другой рынок:
1. Скопируйте этот файл → `market-config.md` в workspace (или создайте `market-config-<region>.md` и укажите агенту).
2. Замените: §1 источники, §2 запросы, §3 зарплатный якорь, §4 работодателей, §5 каналы.
3. Остальные плейбуки остаются без изменений — они читают рынок отсюда.

Локаль интерфейса (`profile/.locale`) при этом может быть любой.
