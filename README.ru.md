# Inbound Career Agent (Агент входящего карьерного позиционирования)

Саморазворачивающийся **инструментально-нейтральный набор навыков (skill pack)**, который превращает вашего кодинг-агента (Claude Code, открытые модели и т. п.) в **ассистента по входящему карьерному позиционированию** — помогает старшим специалистам сделать так, чтобы *нужные* люди сами приходили к ним, с совпадающими целями, вместо массовой рассылки откликов.

> **Статус:** v1, сборка в процессе (P0). Первый рабочий пример — Senior UX/UI Product Design Lead (Санкт-Петербург). Сам набор **универсален**: ваше позиционирование формируется из вашего же онбординга, а не зашито в репозиторий.

## Что делает
- **Сигнал** — делает вас находимым и привлекательным для нужной аудитории.
- **Сенс** — анализ рынка: где спрос, созвучный вашим целям; зарплатные ориентиры вашего уровня.
- **Скрин** — отбор входящих запросов на соответствие вашим целям, прежде чем тратить время.
- Активный поиск работы — **вторичная fallback-функция**, а не основной цикл.

## Как устроено
Один оркестратор (агент, которого вы запускаете) + 7 плейбуков (читаются по необходимости). **Только API** (без скрапинга). **Человек в контуре** (только черновики; вы утверждаете всё сами). Вывод — на вашем языке (по умолчанию английский; доступен русский).

## Три способа запуска
| Способ | Для кого | Как |
|---|---|---|
| **Зелёный** | Пока не готовы работать с агентами | Вставьте `LAUNCHER.md` своему агенту и дайте ссылку на репозиторий. Он всё настроит и проведёт онбординг. |
| **Обычный** | Продвинутые | `git clone`, затем укажите агенту на репозиторий и запустите `SETUP.md`. |
| **Через агента** | Тем, кто даёт агентам ссылки на репозитории | Дайте агенту URL репозитория + `AGENTS.md`; дальше он сам. |

Нет git? См. `git-helper/`.

## Структура репозитория
- `BOOTSTRAP.md` — читать первым (что / зачем / правила).
- `AGENTS.md` — руководство для агента.
- `SETUP.md` — процедура создания рабочего пространства.
- `LAUNCHER.md` — зелёный запуск.
- `AgentContract.xml` — определения ролей и инструментов.
- `TOOLING.md` — настройка провайдера поиска (Tavily, DeepSeek MCP).
- `playbooks/en/`, `playbooks/ru/` — рунбуки по компетенциям по локали (EN по умолчанию).
- `templates/en/`, `templates/ru/` — пустые шаблоны по локали, копируемые в ваше рабочее пространство.
- `market-config.md` — карта источников рынка (RU по умолчанию, заменяемо).
- `claude-code-adapter/`, `git-helper/` — необязательные помощники.

## Дорожная карта
- **P0** репозиторий + документы запуска + заглушка контракта ← текущий этап
- **P1** плейбуки Ритуал A (интейк) + Ритуал B (экспликация)
- **P2** семь плейбуков компетенций
- **P3** подключение инструментов (Tavily, DeepSeek MCP) в `AgentContract.xml`
- **P4** слой рыночных данных
- **P5** адаптер Claude Code + шлифовка + сквозное тестирование
